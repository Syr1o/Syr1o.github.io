PinPoint全链路监控安装及使用

发表于： 2021-08-27
字数：1979 阅读：≈ 4分钟 浏览：

PinPoint简介

PinPoint是开源在GitHub上一款由JAVA语言编写的APM(Application Performance Management)监控工具，用于大规模分布式系统监控。它对性能的影响较小（只增加约 3％资 源利用率），且安装Agent是无侵入式的。

提供的功能

  • 服务映射：通过可视化其组件如何互连来了解任何分布式系统的关联关系。单击节点可显示有关组件的详细信息，例如其当前状态和事务计数。
  • 实时的活跃线程图：监控应用内活跃线程的执行情况，对应用的线程执行性能可以有比较直观的了解。
  • 请求/响应散点图：对分布式环境中每个请求提供了代码维度的可见性，可以在页面中查看请求针对到代码维度的执行详情，帮助查找请求的瓶颈和故障原因。
  • 调用栈：查看有关应用程序的其他详细信息，例如 CPU 使用率，内存/垃圾收集，TPS 和 JVM 参数。

架构的组成

  • pinpoint-agent：用来收集单个应用的信息，并将收集好的应用信息发送到pinpoint-collector 中。
  • pinpoint-collector：用来处理 pinpoint-agent 发送过来的信息，并将信息收集好之后存储到 HBase 中。
  • pinpoint-webUI：查找出 HBase 中的数据并展示。

环境配置及准备

tips：

常规来说需要准备两台虚拟机
一台用于安装pinpoint-collector、pinpoint-webUI、HBase
一台用于用于安装 pinpoint-agent以及承载项目
当然像我这样只使用一台虚拟机也是可以哒，以下所有操作都于linux环境下完成～

JDK配置

pinpoint是由java语言开发，故在安装之前需要安装配置jdk的环境，如果系统已经配置过jdk可跳过～

tar -zxvf jdk-8u231-linux-x64.tar.gz -C /opt/		# 将jdk的tar包指定解压到/opt目录下

vim /etc/profile		# 读写系统配置文件

# 将以下内容复制粘贴至末尾
# set java environment
JAVA_HOME=/opt/jdk1.8.0_231			# jdk文件夹所在路径
JRE_HOME=$JAVA_HOME/jre
PATH=$PATH:$JAVA_HOME/bin:$JRE_HOME/bin
CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar:$JRE_HOME/lib
export JAVA_HOME JRE_HOME PATH CLASSPATH

source /etc/profile		# 加载profile文件

java -version		# 验证是否配置成功

安装包准备

  • HBase下载
    • 下载与pinpoint对应的版本，我Pinpoint版本是2.2.2所以我这里下载的HBase是1.2.7
  • HBase脚本下载
    • hbase-create.hbase
  • pinpoint组件下载
    • pinpoint-web-boot-x.x.x.jar
    • pinpoint-collector-boot-x.x.x.jar
    • pinpoint-agent-x.x.x.tar.gz

安装HBase

我们需要用到HBase数据库来保存监控过程中产生的数据

  1. 把下载的 HBase 安装包上传到 linux 服务器上，并放到指定目录/opt/pinpoint/下面

2

cd /opt/pinpoint

安装PinPoint

pinpoint-collector

  • 以后台运行的方式启动 pinpoint-collector-boot-2.2.2.jar

    1

    nohup java -jar -Dpinpoint.zookeeper.address=localhost pinpoint-collector-boot-2.2.2.jar > collector.log 2>&1 &

  • 持续查看日志信息

    1

    tail -f collector.log

pinpoint-web

  • 以后台的方式运行：pinpoint-web-boot-2.2.2.jar

    1

    nohup java -jar -Dpinpoint.zookeeper.address=localhost pinpoint-web-boot-2.2.2.jar > web.log 2>&1 &

  • 之后可以在浏览器输入地址：http://192.168.255.xxx:8080/ 查看页面，此时还没有agent。

部署 pinpoint-agent 采集监控数据

tips： pinpoint-agent需要与被测应用安装在一起

  1. 基本配置部署

    • 拷贝 pinpoint-agent-2.2.2.tar.gz 到 web 应用服务的目录下

    • 解压并修改配置文件pinpoint-root.config

      1
2
3
4
5

      vim pinpoint.config

profiler.collector.ip=192.168.xxx.xxx  #hbase服务器地址
profiler.transport.grpc.collector.ip=192.168.xxx.xxx   # 安装pinpoint-collector的IP地址
profiler.sampling.counting.sampling-rate=1 # 采样率配置默认是20，1为捕捉每一个请求，访问量不大的情况建议全部捕捉

  2. Tomcat监控

    • 修改测试项目下的 tomcat 启动文件catalina.sh（linux）、catalina.bat（windows），修改这个只要是为了监控测试环境的 Tomcat。

    • 添加探针

      1
2
3
4
5
6
7

      vim catalina.sh

# 添加以下配置
AGENT_PATH=/opt/pinpoint/pinpoint-agent-2.2.2		# pinpoint-agent的目录
AGENT_ID=123456		# agent的ID，唯一且不可重复
APPLICATION_NAME=item_name		# 采集项目的名字，这个名字可以随便取，只要各个项目不重复就好了
VERSION=2.2.2		# agent 的版本。

    • 启动测试用的Tomcat服务器

    • 在pinpointweb浏览器中查看

  3. SpringBoot监控

    • 如果是jar包部署，直接在启动命令中添加启动参数

      1
2

      java -javaagent:/opt/pinpoint/pinpoint-agent-2.2.2/pinpoint-bootstrap-2.2.2.jar
-Dpinpoint.agentId=registry_windows -Dpinpoint.applicationName=item_name -jar item_name.jar

      tips：applicationName – 同一个项目建议相同；agentId – 不同的 jar 包自定义不同的 id。

虚拟机重启后起程序的顺序：Hbase => pinpoint-collector => pinpoint-web => agent-tomcat 或 jar 包
