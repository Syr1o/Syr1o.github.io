基于Github+Hugo搭建个人Blog

发表于： 2020-12-11
字数：1660 阅读：≈ 4分钟 浏览：

搭建博客前

碎碎念

本篇记录于os系统搭建基于Github Pages个人Blog的过程，Windows系统也可作为参考

以往没有用博客记录的习惯，更多时候都是随手拈来的txt或物理载体，导致内容过于分散翻阅成本较高，想开一个博客用于记录平时遇到的问题和学习内容；由于大概率是个人流水账，不想公开于平台，就萌生了自己搭一个的想法；对比了Hexo和Hugo，选择了后者；主题简洁大方，搭建好环境以后，可以使用MarkDown来编写Blog，并且上传到Github上托管，十分便捷。

Hugo简介

Hugo是一个用Go语言编写的动态网页生成器，只须要一个命令 Hugo 就能够在几秒钟内生成一个动态的博客页面，被称为世界上最快的网站构建框架，使Hugo称为最受欢迎且最热门的动态网站生成器之一。（这段为cv产物）

Hugo安装及配置

安装Golang

  • 由于Hugo依赖于Go语言，所以这里需要先部署语言环境：

    1.在Golang官方页面下载页面 下载链接 下载Golang安装包并进行安装

    2.安装后在终端敲命令 go version 若出现相关版本信息，则为安装成功

设置GOPROXY为国内源

  • Go原生配置的GoProxy非常非常慢，所以需要切换成国内的代理源才能获得正常的下载速度

    1. 这里参考 goproxy.io 页面，我们需要在~/.zshrc 中的末尾添加下面这行内容：
    1

    export GOPROXY=https://proxy.golang.com.cn,direct

    1. 随后运行 source ~/.zshrc刷新配置文件

    2. 接着使用go env|grep GOPROXY 确认该配置是否已生效

安装Hugo

  • 由于brew无法正常拉取到代码，所以我这里改用git clone的方式（需要系统安装了Git）：

    # 拉取Hugo代码

# 进入Hugo代码根目录
cd hugo

# 输入指令进行安装
CGO_ENABLED=1 go install --tags extended

# 检查安装是否成功
hugo version

    在命令行中执行hugo help可以查看到hugo的基本用法

构建博客

  • 在本地生成站点（网站的开发目录）

    1. 使用Hugo快速生成初始站点：

      以生成到 /path/to/Hugo_Blog 路径为例，Hugo_Blog为生成的站点目录

      1

      hugo new site /path/to/Hugo_Blog

    2. 新建文章

      在站点目录下运行hugo new hello-world.md将自动生成到content/hello-world.md

      Md头部格式：

      1
2

      ---
title: 文章标题
date: 文章编写时间（请务必按照RFC3339格式的时间进行编写，形如2019-04-15T00:00:00+08:00）
toc: true # 打开文章的目录功能
tags:
 - python
---

文章正文内容
      • 这里注意md文档需要放在站点目录下/content/post/路径下才会生效显示在页面中

    3. 生成页面

      1. 本地站点目录下的config.yaml为配置文件，可以对其进行修改页面功能及布局结构

      2. 运行 hugo server，成功运行后为本地1313端口，可通过localhost:1313 进行访问

Github Page部署

这一步是将网页托管到自己的github或gitee(这里以github为例)，这样就不再是单一的本地访问模式了，这部分需要使用到github提供的一个网页寄存服务Github Page，可以存放静态页面的特性很适合博客搭建。

创建博客代码仓库

  • 在Github上创立一个 Repository，命名为github用户名.github.io

建立关联上传文件

  • 将站点目录下config.yaml文件中的baseURL改为本人博客网址后保存退出，并生成静态页面

    1

    baseURL = "https://xxxxxxxx.github.io/"

  • 执行命令hugo生成静态页面后站点目录下生成了一个public文件夹，里面存放了静态页面的文件，进入public文件夹使用git上传发布

    • 首次上传

    hugo		# 生成静态页面
cd public

git add .		# 添加所有文件
git commit -m "first commit"		# 提交事务,引号内容为该次提交的备注
git push -u origin master			# 推到仓库的master分支
