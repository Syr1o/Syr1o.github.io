Android SDK安装及配置

发表于： 2021-03-04
字数：1997 阅读：≈ 4分钟 浏览：

引入简介

Android SDK

SDK (Software Development Kit )软件开发工具包是软件开发工程师用于为特定的软件包、软件框架、硬件平台、操作系统等建立应用软件的开发工具的集合。 Android SDK 就是 Android 专属的软件开发工具包。

Android SDK是由 JAVA 进行编写的，所以需要先安装JDK 5.0及以上版本。

ADB

ADB 全称为安卓调试桥Android Debug Bridge，是Android提供的一个通用的调试工具，是一个C/S的命令行debug工具，通过这个工具，使得我们的PC能够和Android设备来进行通信。

安装配置

JDK安装及其环境变量配置这里便不再赘述

  1. 首先 前往下载 找到与操作系统版本对应的SDK Tools包，并解压到操作系统任意目录。

  2. 补充SDK功能内容， 前往下载 同样自己系统的platform-tools包，解压后将其移动到上面解压好的目录下

  3. 配置环境变量

    • 配置ANDROID_HOME并设置到该SDK的主目录

    • 将ANDROID_HOME目录下的tools和platform-tools添加到PATH环境变量中

      1

      ;%ANDROID_HOME%\build-tools\android-4.4W;%ANDROID_HOME%\platform-tools;%ANDROID_HOME%\tools;

# 通过以下命令进行sdk的安装
brew install android-sdk

# 安装成功返回
android-sdk was successfully installed!

# 查看安装路径
brew list android-sdk
# 默认为/usr/local/Caskroom/android-sdk/

# 配置环境变量
vim ~/.zshrc

# android sdk 环境变量
export ANDROID_HOME="/usr/local/Caskroom/android-sdk"  # 这个是上面获取到的sdk的安装路径
export PATH="$PATH:$ANDROID_HOME/tools"
export PATH="$PATH:$ANDROID_HOME/platform-tools"

# 保存后使配置生效
source ~/.zshrc

# 安装adb（android-platform-tools）
brew install android-platform-tools

# 安装好后查看是否配置成功
adb version

tips：

这里踩过一个通过brew安装引发的坑，当时在终端我的adb是可以正常使用的，但在后续连接appium的时候抛异常提示说找不到adb工具，看了报错信息得知appium是从环境变量配置的android-sdk目录下去找adb的，但通过brew安装的android-platform-tools路径与android-sdk处于同级目录，将platform-tools移动到sdk文件夹下与另一个tools文件夹同级就可以解决啦。

目录结构详解

  • SDK Manager.exe sdk管理工具，用于管理、下载sdk、sdk工具，能及扩展工具

  • tools 包括测试、调试、第三方工具。模拟器、数据管理工具、ddms等

  • build-tools

    • aapt（Android Asset Packaging Tool）：编译应用程序的资源文件（包括AndroidManifest.xml和你的Activities的xml文件），生成R.java文件，这样你就可以从你的java代码中引用资源
    • aidl：把.aidl 接口转换成java接口，用于优化APK
    • dx：转化.class中间代码为dvlik中间代码,所有经过java编译的生成.class文件都需要此工具进行转换，最后打包进apk文件中

  • platform-tools 保存着一些Android平台相关通用工具，比如adb、和aapt、aidl、dx等文件，这里和platforms目录中tools文件夹有些重复，主要是从android2.3开始这些工具被划分为通用了。Fastboot 刷机工具

  • platform 是每个平台的SDK真正的文件，存放了不同版本的android系统。里面会根据APILevel划分的SDK版本

  • SDK manager安装选择界面

ADB常用命令

  • adb start-server 启动adb服务

  • adb kill-server 关闭adb服务

  • adb devices 查询连接的移动设备

  • adb connect 设备名称:端口号 连接指定Android设备

    1

  • adb -s 设备名称:端口号 操作指定的Android设备

  • adb shell 进入Android系统，可以使用部分的Linux命令

    • exit 退出

    • adb shell 命令 在不进入Android系统情况下，执行该条命令

  • 文件的上传与下载

    • 上传：adb push 本地文件 Android系统目录
    • 上传：adb push 本地文件 Android系统目录

  • 应用的安装与卸载

    • 安装：adb install apk文件

      • -r 覆盖安装

    • 卸载：adb uninstall 包名

      • 获取包名1：adb shell ls /data/data 列出系统当前已经安装的所有应用包名

      • 获取包名2：adb shell pm list packages -3 列出系统当前已经安装的第三方应用包名

      • 获取包名3：aapt dump badging apk文件

      • 获取包名4：

        • 运行应用程序
        • 通过adb shell进入Android系统
        • 执行命令：`dumpsys window windows | grep -i current

  • 应用的启动和停止

    • adb shell input text 文本内容 实现在光标处输入指定内容，只支持英文

    • adb shell input keyevent keycode 发送系统按键

      • keycode对照表

    • adb shell input tap 横坐标 纵坐标 模拟点击

      • 获取坐标1:
        • 运行应用程序，打开想要获取坐标的界面
        • 运行命令adb shell uiautomator dump生成1个xml文件
        • 将xml文件下载到本地，使用浏览器打开即可获取指定元素坐标，bounds=“[910,48] [994,136]”
      • 获取坐标2：设置—关于（使劲点版本号）—开发者选项—指针位置

    • 滑动屏幕 adb shell input swipe 起始x 起始y 终点x 终点y 滑动时间

      • 例：adb shell input swipe 986 906 986 200 5000 实现屏幕向上滑动
